# Max Heap을 활용한 실생활 우선순위 시스템 구현

## 1. 과제 개요

Max Heap을 직접 구현하고, 이를 활용하여 실생활의 우선순위가 중요한 문제를 해결하는 시스템을 만들어봅니다.

하나의 시나리오를 직접 구성한 뒤, 그 문제를 우선순위 큐(priority queue)로 해결할 수 있도록  
<데이터 구조 설계 → 우선순위 계산 → max heap 삽입/삭제 → 시뮬레이션>  
까지 구현하면 됩니다.

## 4. 구현 지침

### 구조체 설계 (예시)

아래 예시는 참고용이며, 여러분의 시나리오에 맞게 자유롭게 수정하세요.

Order 구조체 예시:

- id
- customerLevel
- waitingMinutes
- distanceKm
- priority

필드명은 자유롭게 설정해도 됩니다.  
priority는 여러분이 정의한 계산식에 따라 main.c 내에서 직접 계산해야 합니다.

### 우선순위 계산식 설계

예시 계산식:

- priority = customerLevel _ 50 + waitingMinutes _ 2 - distanceKm \* 1

여러분의 시나리오에 맞는 가중치(weight)와 계산식을 반드시 만들어야 하며,  
보고서에서 “왜 이런 식을 선택했는지”를 설명해 주세요.

### Max Heap 직접 구현

필수 기능

- insert(): 요소 삽입 + 상향 이동
- pop(): 최우선 요소 반환 + 하향 이동

비교 기준은 priority 값입니다.

#### 라이브러리 priority queue는 사용하지 마세요.

## 6. 채점 기준

총 3개의 항목을 평가합니다.

### 1) 시나리오가 Max Heap 문제로서 타당한가?

- 우선순위 큐가 필요한 문제인지
- 구조체 설계와 우선순위 식이 논리적인지

### 2) 10개 이상의 노드를 삽입/pop하는 테스트에서 정확하게 동작하는가?

- max heap의 삽입/삭제(percolate up/down)가 정확한지
- 우선순위대로 pop 되는지
- 전체 프로그램이 안정적으로 동작하는지

### 3) 보고서와 코드의 내용이 일치하는가?

- 구조체 필드 설명이 코드와 동일한지
- 설명한 우선순위 계산식이 코드에도 동일하게 구현되었는지
- 출력 결과가 보고서의 설명과 맞는지
